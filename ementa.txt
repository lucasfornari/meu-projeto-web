Desafio Prático: Lançamento do Projeto Web (Git + Bootstrap)
Objetivo: Criar uma página HTML básica, integrar o framework CSS Bootstrap usando branches isoladas, mesclar o trabalho e resolver um conflito de código proposital.
Passo 1: O Esqueleto do Projeto
Crie uma pasta chamada meu-projeto-web no seu computador e inicialize um repositório Git vazio dentro dela.
Crie um arquivo chamado index.html e insira nele a estrutura básica de um documento HTML5.
Adicione o arquivo ao controle de versão (staging area) e faça o seu primeiro commit com uma mensagem descritiva (exemplo: "docs: cria estrutura basica do html").
Passo 2: O Desenvolvedor Front-end Entra em Ação
Crie uma nova ramificação (branch) chamada adiciona-estilo e mude para ela.
No arquivo index.html, adicione o link do CDN do Bootstrap dentro da tag <head>.
Salve o arquivo e faça o commit dessa melhoria na sua branch atual.
Passo 3:  Seguindo
Volte para a branch principal (main ou master). Dica: Ao fazer isso, o link do Bootstrap deve desaparecer do seu código.
Dentro da tag <body>, adicione um título simples usando a tag <h1> (exemplo: "Página em Construção").
Salve o arquivo e faça o commit dessa adição na branch principal.
Passo 4: O Primeiro Merge
Certifique-se de que você está na branch principal.
Traga as alterações feitas na branch do design (adiciona-estilo) para a sua branch atual através de uma mesclagem (merge).
Abra o arquivo index.html no seu navegador. A fonte do título deve estar diferente, indicando que o Bootstrap agora está aplicado à versão principal!
Passo 5: O Caos Controlado (Gerando um Conflito Proposital)
Dois membros da equipe resolvem alterar a mesma linha de código ao mesmo tempo em ramificações diferentes.
Crie uma nova branch chamada nova-ideia e mude para ela.
Mude o texto dentro do seu <h1> para uma nova frase (exemplo: "Bem-vindo ao Nosso Novo Produto") e faça o commit.
Agora, volte para a branch principal (main).
Nesta branch, modifique o mesmo <h1>, mas coloque um texto diferente e adicione algumas classes de estilo do Bootstrap (exemplo: <h1 class="text-center text-primary mt-5">Lançamento Oficial</h1>).
Faça o commit dessa nova alteração na branch principal.
Passo 6: A Resolução do Conflito
Ainda na branch principal, tente fazer o merge da branch nova-ideia. O Git deverá emitir um alerta de conflito (Merge conflict).
Abra o seu arquivo index.html no editor de código. Você verá as marcações automáticas que o Git insere para mostrar as duas versões que colidiram.
Sua missão: Resolva o conflito limpando as marcações do Git e unindo o melhor das duas versões. O código final deve manter as classes de estilo do Bootstrap, mas o texto do título deve ser uma combinação das duas ideias (ex: "Lançamento Oficial: Bem-vindo ao Nosso Novo Produto").
Após editar e salvar, diga ao Git que o conflito foi resolvido adicionando o arquivo novamente e criando um commit para finalizar a mesclagem.
Passo 7: Admirando o Histórico
Utilize o comando do Git que permite visualizar todo o histórico de commits em formato de gráfico, exibindo os caminhos em uma única linha e mostrando todas as ramificações.
git log --oneline --graph --all

📝 Entregáveis para Avaliação
Um print screen da página index.html aberta no navegador (deve mostrar o título combinado com o estilo do Bootstrap aplicado).
Um print screen do terminal executando o comando final do Passo 7, mostrando o gráfico com os commits, a separação das branches e os merges realizados.
Repositório no GITHUB. Enviar link para Professor


Desafio Prático Parte 2: Expandindo o Projeto Web (Multi-páginas com Bootstrap)
Objetivo: Evoluir a página única para um site completo (Home, Produtos e Contato) utilizando componentes de layout padrão do Bootstrap e consolidar o fluxo de trabalho com branches (ramificações) no Git.

Link Bootstrap: https://getbootstrap.com/docs/5.2/getting-started/introduction/ 
Passo 8: Preparando o Terreno (Nova Branch)
Abra o terminal no seu repositório meu-projeto-web.
Certifique-se de que está na branch main e que não há alterações pendentes.
Crie e mude para uma nova branch chamada desenvolvimento-site (ex: git checkout -b desenvolvimento-site).
É nesta ramificação isolada que você construirá as novas páginas.
Passo 9: A Barra de Navegação (Navbar)
Abra o seu index.html (que agora é a sua página Home).
Logo após a tag <body>, adicione um componente Navbar padrão do Bootstrap.
Configure os links da Navbar para apontar para três arquivos: index.html (Home), produtos.html (Produtos) e contato.html (Contato).
Mova o título <h1> que você criou no desafio anterior para dentro de um contêiner Bootstrap (<div class="container">) abaixo da Navbar.
Salve e faça o commit (ex: "feat: adiciona navbar no index").
Passo 10: A Vitrine de Produtos
Crie um novo arquivo chamado produtos.html na mesma pasta.
Copie a estrutura básica do seu index.html (incluindo o CDN do Bootstrap no <head> e a Navbar que você acabou de criar).
Abaixo da Navbar, crie um <div class="container">.
Utilize o sistema de Grid e os Cards do Bootstrap para exibir 3 produtos fictícios lado a lado (dica: use as classes row e col-md-4, e adicione um componente Card em cada coluna com imagem de placeholder, título e botão de compra).
Adicione o arquivo ao stage e faça o commit (ex: "feat: cria pagina de produtos com cards bootstrap").
Passo 11: A Página de Contato
Crie um novo arquivo chamado contato.html.
Novamente, copie a estrutura básica com o CDN do Bootstrap e a Navbar.
Dentro de um <div class="container mt-5">, adicione um formulário utilizando os estilos padrão de Forms do Bootstrap.
O formulário deve conter:
Um campo de texto para o Nome.
Um campo de e-mail (tipo email).
Uma área de texto (textarea) para a mensagem.
Um botão de envio estilizado (ex: <button class="btn btn-success">Enviar</button>).
Adicione e faça o commit (ex: "feat: cria formulario de contato").
Passo 12: O Grande Merge e Publicação
O seu site agora está completo na branch desenvolvimento-site. Volte para a branch main.
Faça o merge da branch desenvolvimento-site para a main (git merge desenvolvimento-site). Como você editou arquivos novos e não mexeu em linhas conflitantes, o merge deve ocorrer de forma limpa (Fast-forward).
Conecte seu repositório local a um repositório vazio no GitHub (caso ainda não tenha feito na etapa anterior) e envie o código usando git push -u origin main.

Desafio Prático Parte 3: Interatividade com JavaScript (Simulador de Orçamento)
Objetivo: Introduzir a lógica de programação com JavaScript para manipular o DOM (Document Object Model), realizando cálculos em tempo real baseados nas interações do usuário, e fixar o fluxo de criação de features em branches isoladas.
Passo 13: Preparando a Interface (HTML + Bootstrap)
No seu terminal, garanta que está na branch main e atualizada. Crie uma nova branch para esta funcionalidade: git checkout -b feature-orcamento.
Abra o arquivo produtos.html. Vamos transformar os "Cards" de produtos em itens selecionáveis.
Dentro de cada Card de produto, adicione:
Um Checkbox (caixa de seleção) para o cliente marcar se quer o produto. Dê a cada checkbox um atributo value com o preço do produto (ex: value="50.00") e uma classe comum, como class="item-produto".
Um Input numérico (<input type="number">) para a quantidade, com valor inicial 1 e min="1". Dê uma classe como class="qtd-produto".
No final da página, antes do rodapé (ou abaixo dos produtos), crie um contêiner destacado com Bootstrap (ex: uma div com as classes alert alert-info text-center) contendo um texto grande: "Valor Total: R$ <span id="valor-total">0,00</span>". É aqui que o JavaScript vai injetar o resultado.
Salve o arquivo e faça o commit (ex: "feat: atualiza layout de produtos para simulador").
Passo 14: A Mágica da Interatividade (JavaScript)
Crie um novo arquivo na sua pasta chamado script.js.
Volte ao produtos.html e, logo antes da tag de fechamento </body>, adicione a chamada para o seu script: <script src="script.js"></script>.
No arquivo script.js, escreva uma função (ex: calcularTotal()) que faça o seguinte:
Selecione todos os checkboxes dos produtos e seus respectivos inputs de quantidade.
Crie uma variável total iniciando em 0.
Faça um laço de repetição (for ou forEach) que passe por cada checkbox. Se ele estiver marcado (checked), multiplique o valor dele (value) pela quantidade correspondente e some ao total.
Selecione o elemento HTML onde o total será exibido (document.getElementById('valor-total')) e atualize seu texto (innerText ou textContent) com o valor calculado.
Para que o cálculo seja automático, adicione "ouvintes de eventos" (Event Listeners) aos checkboxes e aos inputs de quantidade para que, sempre que houver uma mudança (change), a função calcularTotal() seja executada novamente.
Salve os arquivos produtos.html e script.js, adicione-os ao stage e faça o commit (ex: "feat: adiciona logica JS para calculo de orcamento").
Passo 15: Integrando a Solução e Publicando
O seu simulador agora deve estar funcionando na branch feature-orcamento. Teste no navegador! Marque os produtos, mude as quantidades e veja o valor total atualizar sozinho.
Tudo certo? Volte para a branch principal: git checkout main.
Traga a funcionalidade finalizada para a sua base de código fazendo o merge: git merge feature-orcamento.
Sincronize seu projeto local com o GitHub executando: git push origin main.


Desafio Prático Parte 4: Consumindo uma API Externa (O Mundo Dinâmico)
Objetivo: Aprender a buscar dados reais da internet usando a Fetch API do JavaScript, transformar esses dados (JSON) em elementos HTML dinâmicos na tela usando o Bootstrap, e manter as boas práticas do Git.
Passo 16: O Ambiente Seguro (Nova Branch)
Abra o terminal, garanta que está na branch main e crie a ramificação para a nova funcionalidade: git checkout -b feature-depoimentos.
O objetivo aqui será criar uma seção de "Depoimentos de Clientes" na página inicial, gerada automaticamente a partir de dados da internet.
Passo 17: Preparando o HTML na Home
Abra o seu index.html. Abaixo do conteúdo principal, crie um novo contêiner (<div class="container mt-5">) com um título <h2> chamado "O que nossos clientes dizem".
Dentro desse contêiner, crie uma <div class="row" id="lista-depoimentos"></div>. Ela estará vazia no HTML, pois o JavaScript é quem vai preenchê-la.
Salve e faça o commit (ex: "feat: cria container para depoimentos dinamicos").
Passo 18: Buscando os Dados com JavaScript
Abra o seu arquivo script.js.
Crie uma função assíncrona (ex: async function carregarDepoimentos()).
Dentro dela, use o comando fetch() para buscar dados de uma API pública e gratuita de testes (exemplo: https://jsonplaceholder.typicode.com/comments?_limit=3).
Converta a resposta para JSON.
Faça um laço de repetição (forEach) nos dados recebidos. Para cada item, crie dinamicamente um "Card" do Bootstrap usando template strings (crases: `) e insira-o dentro da div com o id lista-depoimentos usando innerHTML.
Chame a função carregarDepoimentos() no final do seu script para que ela rode assim que a página carregar.
Teste no navegador! Você deve ver 3 cards gerados dinamicamente com dados da API.
Salve, adicione ao stage e faça o commit (ex: "feat: consome API e renderiza depoimentos").
Passo 19: Unindo as Peças
Volte para a branch principal: git checkout main.
Faça o merge da sua funcionalidade: git merge feature-depoimentos.
Envie para o repositório remoto: git push origin main.

Desafio Prático Parte 6: O Modo "Hard" (APIs, Módulos e Componentes Dinâmicos)
Passo 20: Comunicação de Via Dupla (Enviando Dados com Fetch API)
Objetivo: Evoluir o uso da Fetch API. Em vez de apenas ler dados (GET), o aluno deverá enviar os dados do formulário de contato para um servidor (POST), sem recarregar a página, simulando o comportamento de aplicações modernas (SPAs).
Ações:
A partir da main, crie a branch: git checkout -b feature-envio-form.
No arquivo script.js, dentro da validação do formulário de contato, capture os valores digitados nos campos (Nome, Email e Mensagem).
Agrupe esses dados em um objeto JavaScript e converta-o para texto usando JSON.stringify().
Use a Fetch API para enviar esses dados para o site JSONPlaceholder (ex: https://jsonplaceholder.typicode.com/posts). Para isso, adicione um objeto de configuração no fetch contendo: method: 'POST', os headers adequados ('Content-type': 'application/json') e o seu JSON no body.
Use async/await para aguardar a resposta. Se a requisição der certo (status 201), use JavaScript para injetar um alerta verde (sucesso) do Bootstrap na tela. Se falhar, injete um alerta vermelho.
Salve, faça o commit (ex: "feat: envia dados do form via POST") e faça o merge para a main.
Passo 21: Dividir para Conquistar (Modularização com ES6)
Objetivo: O arquivo script.js acumulou validações, cálculos, GET e POST. O desafio agora é refatorar o código, separando as responsabilidades usando import e export nativos do JavaScript, exatamente como profissionais fazem.
Ações:
Crie a branch: git checkout -b refatoracao-modulos.
No arquivo HTML, modifique a tag do script para ativar os módulos: <script type="module" src="js/main.js"></script>.
Crie uma pasta chamada js e divida seu código atual em três arquivos:
api.js: Conterá apenas as funções de comunicação com a internet (o fetch de depoimentos e o envio do form). Use export nestas funções.
ui.js: Conterá apenas a manipulação do DOM (injetar alertas do Bootstrap, renderizar cards). Use export aqui também.
main.js: Será o arquivo principal. Ele importará as funções (import { ... } from './api.js') e fará a ligação com os eventos de clique e envio (Event Listeners).
Atenção: Módulos ES6 exigem um servidor local para rodar (devido ao CORS). Abra o projeto usando a extensão Live Server do VS Code para testar.
Faça o commit (ex: "refactor: separa logica em modulos ES6") e mescle com a main.
Passo 22: Reutilização de Componentes (Modal Dinâmico do Bootstrap)
Objetivo: Evitar a criação de uma página HTML para cada produto. O aluno usará um único "Modal" (janela flutuante) do Bootstrap e usará JavaScript para injetar o conteúdo correto dependendo de qual botão foi clicado.
Ações:
Crie a branch: git checkout -b feature-modal-dinamico.
No produtos.html, cole o código HTML de um Modal vazio do Bootstrap (apenas a estrutura, sem textos) no final da página, antes do fechamento do <body>.
Nos botões "Ver Detalhes" dos cards de produtos, adicione atributos de dados customizados (data attributes) com as informações daquele produto específico (ex: data-nome="Teclado Gamer", data-descricao="Teclado mecânico RGB", data-preco="200.00").
No seu JavaScript (ui.js ou main.js), adicione um ouvinte de evento global para a abertura do modal: document.getElementById('seuModal').addEventListener('show.bs.modal', function (event) { ... }).
Dentro da função, identifique qual botão acionou o modal usando event.relatedTarget. Leia os atributos data-* desse botão e injete dinamicamente esses textos no Título e no Corpo do modal vazio antes de ele aparecer na tela.
Faça o commit final, volte para a main, faça o merge e envie tudo para o GitHub (git push origin main).

Desafio Prático Parte 7: Levando o Projeto ao Nível Profissional (Revisado)
Passo 23: Integração Externa (Auto-preenchimento de Endereço com ViaCEP)
Objetivo: Consolidar o uso da Fetch API fazendo com que o formulário de finalização de orçamento ou contato preencha o endereço do usuário automaticamente ao digitar o CEP. Ações:
A partir da branch main, crie a branch: git checkout -b feature-viacep.
No seu HTML (no modal ou página de formulário), garanta que existam campos de input para CEP, Rua, Bairro, Cidade e Estado.
No JavaScript, adicione um ouvinte de evento blur (acionado quando o usuário clica fora do campo) ou input no campo de CEP.
Assim que um CEP válido for digitado (8 números), faça uma requisição usando fetch(https://viacep.com.br/ws/${cep}/json/).
Pegue a resposta em JSON e crie uma função que preencha automaticamente os valores (input.value) dos campos de Rua, Bairro, Cidade e Estado correspondentes.
Adicione um tratamento de erro (catch ou checagem de erro: true no retorno da API) para alertar o usuário caso o CEP seja inválido.
Faça o commit (exemplo: feat: adiciona busca de cep automatica com viacep) e mescle com a main.
Passo 24: Acessibilidade e UX (Tema Claro/Escuro Dinâmico)
Objetivo: Utilizar os recursos do Bootstrap para adicionar um botão que alterna o site entre o "Modo Claro" e o "Modo Escuro" (Dark Mode), salvando a preferência do visitante. Ações:
Crie a branch: git checkout -b feature-dark-mode.
Na sua Navbar, adicione um botão ou switch (interruptor) com o texto "Alternar Tema".
No seu arquivo de interações da interface (ui.js, modularizado no Passo 21 ), adicione um ouvinte de evento (Event Listener) para o clique deste botão.
Quando clicado, o JavaScript deve alternar o atributo data-bs-theme="dark" (que é o padrão do Bootstrap para temas) diretamente na tag <html> do seu documento.
Aproveite o conhecimento do passo anterior e salve a escolha do usuário (tema: 'dark' ou tema: 'light') no localStorage. Isso garantirá que o tema escolhido permaneça o mesmo enquanto o usuário navega entre as páginas de Home, Produtos e Contato.
Faça o commit (exemplo: feat: implementa toggle de dark mode e salva preferencia) e mescle com a main.
Passo 25: O Grand Finale (Deploy Público com GitHub Pages)
Objetivo: Tirar o site do seu computador (localhost) e publicá-lo na internet de forma gratuita para compor o seu portfólio. Ações:
Como você já conectou o projeto ao GitHub e vem sincronizando sua branch main com git push ao longo do projeto, o código principal já está pronto na nuvem.
Acesse a página do seu repositório no GitHub pelo navegador.
Vá na aba "Settings" (Configurações) e, no menu lateral esquerdo, clique em "Pages".
Na seção "Build and deployment", no campo "Source", escolha "Deploy from a branch".
Logo abaixo, em "Branch", selecione a sua branch main e a pasta / (root). Clique em Save.
Aguarde alguns minutos. O GitHub fará a publicação automaticamente e exibirá um link público no topo da tela (exemplo: https://seusuario.github.io/meu-projeto-web).
Volte ao seu editor de código na branch main, crie um arquivo README.md detalhando as tecnologias usadas no projeto e insira o link do site no ar. Faça o último commit (docs: adiciona readme com link de producao) e envie para o repositório remoto com git push.
